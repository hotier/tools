# 🔥 Next.js 一体化版本&功能更新管理体系

基于 **Changesets** 的一站式版本管理方案，实现：
**功能→变更→版本→发布→更新→回滚 全链路打通**

## 🎯 一站式方案总览

```
1. 开发功能 → 2. 记录功能变更（Changesets）→ 3. 统一聚合版本号
→ 4. 自动发版（改版本号+生成日志+打Git标签）
→ 5. Next.js 客户端检测版本更新
→ 6. 线上问题→一键回滚（按版本Tag）
```

**所有环节在一个体系内，无手动改版本、无零散管理**

---

## 📦 版本号管理（全自动，零手动修改）

### 核心规则：严格 SemVer

- `patch`：bug修复、小优化 → 自动升第三位（v1.0.0→v1.0.1）
- `minor`：新功能、兼容更新 → 自动升第二位（v1.0.1→v1.1.0）
- `major`：不兼容重构 → 自动升第一位（v1.1.0→v2.0.0）

### 版本号生命周期

1. 开发完成 → 记录变更
2. 聚合变更 → 自动计算版本号
3. 发版 → 自动写入 package.json + 打 Git Tag
4. 回滚 → 按 Tag 直接回退版本

## 🧩 功能更新管理（按功能归集，可追溯）

### 日常操作：开发完功能，10秒记录

```bash
npm run change
```

交互步骤：
1. 选择变更类型：`patch/minor/major`
2. 输入功能描述
3. 自动生成 `.changeset/xxx.md` 功能变更文件

### 功能管理优势

✅ 所有功能更新**按文件归集**，不会丢失
✅ 多人协作**互不冲突**，每人记录自己的功能
✅ 发版时**自动合并所有功能**，生成完整更新日志
✅ 可回溯：哪个版本包含哪些功能，一目了然

---

## 🔄 发布&更新流程（全自动化）

### 1. 聚合所有功能，自动生成版本号

```bash
npm run version
```

执行后自动完成：
- 修改 `package.json` 版本号（**零手动**）
- 生成/更新 `CHANGELOG.md`（所有功能更新清单）
- 删除已聚合的 `.changeset` 临时文件

### 2. 一键发版（版本+标签+远程同步）

```bash
npm run release
```

最终产出：
- 版本号：`package.json` 自动更新
- Git Tag：`vX.X.X`（可直接回滚）
- 更新日志：`CHANGELOG.md`（功能更新全记录）
- 远程同步：代码+版本+标签全部推送

## 🔍 客户端更新检测

版本号联动，自动提示新功能，解决缓存问题。

### 工作原理

1. `src/utils/version.ts` - 从 package.json 读取版本号
2. `src/components/UpdateChecker.tsx` - 全局更新检测组件
3. `src/app/api/version/route.ts` - 版本 API

### 自动检测时机

- 页面加载时
- 页面激活时（visibilitychange）

## ⏪ 回滚管理（版本+功能同步回退）

### 按版本Tag回滚

```bash
npm run rollback
```

选择历史版本标签进行回滚。

```bash
# 1. 开发功能
git checkout -b feature/xxx

# 2. 记录功能（写入功能更新）
npm run change

# 3. 聚合版本（自动升版本号）
npm run version

# 4. 一键发版（发布+标签+日志）
npm run release
```

## ✅ 一站式方案核心价值

1. **版本号管理**：全自动，永不手动改，严格SemVer，全局唯一
2. **功能更新管理**：每个功能绑定变更，可归集、可追溯、可合并
3. **版本管控**：Git Tag+Changesets双保险，发布/回滚一键搞定
4. **客户端更新**：版本号联动，自动提示新功能，解决缓存问题
5. **全闭环**：从开发→功能记录→版本→发布→更新→回滚，**一个体系、一套工具、无零散操作**
